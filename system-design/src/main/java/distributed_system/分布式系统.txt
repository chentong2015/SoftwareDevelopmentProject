Distributed Systems Key Points:
- Scalability
  可扩展性是为了满足不断增长的work需求
  可扩展的系统可以在没有系统损失的情况下进行扩展
  Horizontal Scaling 添加更多的服务server到资源池
  Vertical Scaling 提高每个server的capacity

- Reliability
  体现系统核心服务不会被挂掉的能力
  Redundancy of both the software components and data 通过提供副本来实现
  冗余是有成本的，可靠的系统必须付出代价才能通过消除每个单点故障

- Availability
  可用性是指系统在特定时期内保持运行以执行其所需功能的时间
  如果系统是可靠的，则必然是可用的，反之可用不代表可靠

- Efficiency

- Serviceability or Manageability
  how easy it is to operate and maintain ?
  可维护性和可操作性，多快以及多简单能够将系统修好

- Sustainable 系统的可持续性

high performance 高性能 (系统稳定且高效)
high concurrency 高并发 (能够支撑起大量用户的并发访问)
high consistent 一致性 (data should be all same on different devices)
Cost 花销
Security 安全
Monitoring 监控
