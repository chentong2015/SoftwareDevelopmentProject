如何设计软件系统
1. 逐步分析，一步一步找到正确的答案
2. 将复杂的问题查分出来，构建可以重用的类库
3. 当有前端的UI时，一定要考虑用户的使用体验(操作的简化，提示，预操作)
4. 架构设计一定要清晰，方便开发和后续的扩展Extension

软件系统背后的核心：通过分布式系统来解决可用性和扩展性
高可用availability/reliability
高扩展scalability
可维护maintainability
高性能high performance

- system architecture patterns
