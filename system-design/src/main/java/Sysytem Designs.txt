如何设计软件系统
1. 逐步分析，一步一步找到正确的答案
2. 将复杂的问题查分出来，构建可以重用的类库
3. 当有前端的UI时，一定要考虑用户的使用体验(操作的简化，提示，预操作)

软件系统背后的核心：
高可用availability/reliability，高扩展scalability，高性能high performance
花销cost, 实现的复杂度complexity
- standard design principles, patterns, scalability principles
- system architecture patterns
- main components and their interactions: architecture, data models, APIs, and algorithms.
